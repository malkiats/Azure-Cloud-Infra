## What is PowerShell?
PowerShell is a **command-line tool and scripting language** used to **manage systems and automate tasks**.

It helps you control computers, servers, and cloud resources using simple commands called **cmdlets**.

---

## What Can You Do with PowerShell?

- Run commands on **local or remote machines**
- Manage **users, files, and system settings**
- Automate **repetitive tasks**
- Manage **cloud resources** (Azure, AWS, etc.)
- Support **DevOps and CI/CD workflows**
- Write scripts to save time and reduce manual work

---

## Why Use PowerShell?

- Easy to learn and powerful
- Works well for **system admins and developers**
- Supports automation and scripting
- Can be extended using **modules**
- Works on **Windows, Linux, and macOS**

---

## Key Features

- Command-line interface
- Script-based automation
- Object-based output (not just text)
- Supports remote management
- Large community and module ecosystem

---

## Learning Objectives

After learning PowerShell, you will be able to:
- Understand what PowerShell is and how it works
- Run commands to automate daily tasks
- Use PowerShell for system and cloud management

---

## Prerequisites

- Basic knowledge of using a command line (CMD, Terminal, or Git Bash)
- Visual Studio Code installed
- Ability to install software on your system


# Simple Explanation
---

## Why Use PowerShell?

- Run commands instead of clicking through menus
- Automate repeated tasks
- Manage systems, servers, and cloud resources
- Useful for DevOps, IT admins, and developers

---

## What Makes PowerShell Useful?

### 1. Command-Line Interface
You type commands instead of using a mouse.  
This makes work **faster and scriptable**.

### 2. Automation
You can save commands in scripts and run them again anytime.

### 3. Works with Cloud & DevOps
Used to manage:
- Azure resources
- CI/CD pipelines
- Servers and services

### 4. Object-Based (Not Text-Based)
- Commands return **objects**, not plain text  
- Easier to filter, sort, and reuse data

---

## Key Features

- Built-in help system (`Get-Help`)
- Supports command pipelines (`|`)
- Uses **cmdlets** (special PowerShell commands)
- Supports aliases like `ls`, `cls`
- Can be extended using modules

---

## What Are Cmdlets?
- Cmdlets are PowerShell commands
- Example: `Get-Process`, `Get-Service`
- They follow a **Verb-Noun** format
- They return structured data (objects)

---

## Where Can You Use PowerShell?

- Windows (pre-installed)
- Linux and macOS (install separately)
- Azure Cloud Shell
- Visual Studio Code (recommended)
